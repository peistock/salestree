# 技能蒸馏工作流 Skill

## When to Use

当完成用户请求时，任务涉及以下多个维度：
- 六源并行调研（著述、对话、表达、他者视角、决策、时间线）
- 多角度验证和交叉检验
- 提炼核心原则或心智模型
- 构建可操作的框架或方法论
- 创建详细文档并通过质量检验
- 有明确的输出 artifact（如SKILL.md、研究报告等）

## Procedure

### 1. 六源并行调研已完成
- **著述**：收集权威书籍、文章、报告等文字资料
- **对话**：进行访谈、专家咨询、用户反馈等口头交流
- **表达**：分析目标受众的表达方式、沟通风格、偏好
- **他者视角**：考虑不同利益相关者的看法和需求
- **决策**：研究决策过程、影响因素、案例分析
- **时间线**：了解历史发展、趋势演变、时间依赖关系

### 2. 通过三重验证提炼核心内容
- **验证1**：信源可靠性（优先使用Tier 1和Tier 2信源）
- **验证2**：交叉验证（至少2-3个独立信源验证关键点）
- **验证3**：实用性验证（检验提炼内容是否具有操作指导意义）

### 3. 提炼决策启发式或行动框架
- 从复杂信息中提炼出3-5个核心心智模型或原则
- 每个心智模型配以具体的应用场景和操作步骤
- 提炼决策启发式：简单规则或检清单，指导实际决策
- 确保启发式具有广泛适用性而非过度特化

### 4. 完成表达DNA分析（可选但推荐）
- 口语化风格：是否适合口头表达和对话场景
- 幽默元素：适当使用自嘲、讽刺或轻松语调增加亲和力
- 结构偏好：故事类比、类比推理还是直说事实
- 受众匹配：根据目标用户调整表达方式和复杂度

### 5. 构建完整artifact并质量检验
- 创建结构化文档（SKILL.md、报告、指南等）
- 区块清晰：When to Use、Procedure、Pitfalls、Verification、Changelog
- 质量检查项（至少6项通过）：
  - 触发场景描述清晰
  - 操作步骤可执行
  - 已知陷阱列出
  - 验证方法明确
  - 版本记录完整
  - 引用格式规范

### 6. 正确放置和索引
- Skill文件位置：`auto-generated/{category}/{skill-name}/SKILL.md`
- 更新对应分类目录下的index.json
- 在SKILLs/auto-generated/README.md中追加技能列表条目
- 确保文件命名使用kebab-case格式

## Pitfalls

- **不要**跳过验证步骤直接基于单一信源创建技能
- **不要**创建过于笼统或过于具体的技能（需平衡通用性和特指性）
- **不要**忽略用户实际使用场景，只停留在理论层面
- **不要**忘记更新索引文件导致技能不可发现
- **不要**创建重复已有功能的技能（先检查现有技能库）

## Verification

技能创建完成后检查：
- [ ] 是否完成了六源并行调研？
- [ ] 是否通过了三重验证提炼核心内容？
- [ ] 是否提炼出可操作的心智模型或决策启发式？
- [ ] 是否进行了表达DNA分析（针对目标受众）？
- [ ] 是否创建了完整结构化文档并通过质量检验？
- [ ] 是否正确放置并在索引中可发现？
