# AI 项目五维评估框架（销销自检标准）

> 行业通用框架，用于判断 AI 项目是生产工具还是 demo。销销每次重大迭代前必须跑一遍。

---

## 一、五个维度

| 维度 | 核心问题 | 及格线（6/10） | 优秀线（8/10） |
|------|---------|--------------|--------------|
| **1. 基础模型能力** | 模型参数量/推理能力是否匹配任务复杂度？ | 日常任务稳定，不频繁幻觉 | 复杂任务（长报告、多步推理）也能胜任 |
| **2. 上下文工程质量** | 记忆/检索/关联是否超越"简单 RAG + 历史拼接"？ | 有分层记忆 + 向量检索 | 语义关联 + 召回质量可评估 + context 精细管理 |
| **3. 规划与迭代机制** | 任务超过 8-10 步会崩吗？失败能回退吗？ | 有循环 + 有任务跟踪 | 有前瞻/重规划 + 失败自动回退 + 子目标分解 |
| **4. 编排模式** | 并行/路由/链式/编排-执行是否系统化？ | 有并行执行 + 基本路由 | 显式编排框架（Orchestrator-Workers / Prompt Chaining）|
| **5. 可靠性控制** | 输出质量谁评估？胡说八道怎么拦？ | 参数校验 + 基础安全扫描 | Evaluator + Guardrail + Feedback Loop + 自动重试 |

## 二、销销当前得分（2026-07-10）

| 维度 | 得分 | 现状 | 差距 |
|------|------|------|------|
| **1. 基础模型能力** | **7/10** | LM Studio 本地 qwen3.6-35b-a3b 日常任务稳定；已接入 Agnes/DeepSeek/百炼等 API 备选，并按错误类型自动故障转移 | 持续评估更强模型性价比；在真实场景中验证 fallback 稳定性 |
| **2. 上下文工程质量** | **6/10** | 分层记忆 + 关联引擎 + 压缩器已改为 system 角色注入、token 阈值触发、工具输出裁剪 | 关联引擎需语义化；RAG 需评估机制 |
| **3. 规划与迭代机制** | **5/10** | while 循环 + Todo + Checkpoint + Plan 自动推进已有骨架；已注入 Plan-Execute-Verify 三段式循环（首轮提醒制定计划、执行中自动推进、最终 Verify 校验），Evaluator 可自检输出质量并触发重试 | 需在实践中验证重规划能力；复杂任务失败自动回退仍需加强 |
| **4. 编排模式** | **5/10** | 并行工具 + delegate 子 Agent + Skill 路由好，但缺显式 Prompt Chain 和 Orchestrator-Workers | 长任务需系统化编排框架 |
| **5. 可靠性控制** | **6/10** | 前置校验 + risk_scanner + 串行锁 + Guardrail（涉及时效性内容未调用工具时强制重试）+ Evaluator feedback loop + LLM 自动故障转移 + 子 Agent timeout/cancel_event + 消息清理 | Evaluator 评分策略可继续向量化；意图路由可继续向 LLM 分类升级 |

**总体判断**：销销不是 demo，但离"可以放心让销售团队 7x24 依赖"还有明显距离。最大短板仍在 **3（规划）**。

## 三、改进优先级

**P0（下次迭代必须解决）**：
- [x] 模型能力：持续评估更强模型性价比，建立自动降级路由
- [x] 上下文工程：修复压缩器脏数据注入与错误触发时机
- [x] 迭代预算：引入时间/token/迭代三维预算控制

**P1（接下来 2-4 周）**：
- [x] 子 Agent 超时与取消：取消从 pass 改为 `threading.Event`，并行执行加 `future.result(timeout=60)`
- [x] 消息清理：整合 Hermes 风格 sanitization（surrogate/控制字符/图片/工具参数修复）
- [x] 意图路由：去掉长度启发式，改为规则 + 复杂话题优先的兜底策略
- [x] 规划机制：三段式循环（Plan → Execute → Verify）
- [x] 可靠性：Guardrail/Evaluator 结果接入 Feedback Loop

**P2（长期）**：
- 上下文工程：关联引擎从硬编码规则升级为语义关联 + RAG 评估
- 编排模式：Orchestrator-Workers 框架用于复杂报告生成
