# 行业资讯与客户分享

## 目标

从 wechat-digest 资讯库中召回相关文章，生成一段销售可以直接发给客户的行业洞察 / 资讯摘要，帮助销售在跟进中提供价值。

## 执行流程

1. **识别需求**
   - 从用户输入中提取：客户公司名/行业/关注主题、时间范围、输出形式（微信一段 / 邮件 / 朋友圈）。
   - 如果信息不足，先简短反问。

2. **查 CRM（可选）**
   - 如果用户提供客户公司名，用 `get_account` 查客户行业、阶段、备注，把检索词聚焦到和客户业务最相关的方向。

3. **检索资讯库**
   - 用 `search_industry_news` 检索 5-10 条相关文章。
   - 检索词优先用客户行业 + 业务方向，例如「快消品 短视频营销」「汽车品牌 私域运营」。

4. **筛选与综合**
   - 从召回结果里挑 3-5 条最相关、最新、对客户最有价值的文章。
   - 每条提炼成一句话洞察，标注来源（标题、公众号、日期、链接）。

5. **输出分享文案**
   - 生成可直接发送的文案：
     - 开头一句问候/上下文
     - 核心资讯 3-5 条，每条「一句话结论 + 来源」
     - 结尾一句自然引导（如感兴趣可进一步聊/发方案）

6. **收尾动作**
   - 询问是否需要：
     - 生成营销方案（转 marketing-plan）
     - 生成跟进邮件/微信触达文案（转 outreach-drafter）
     - 深挖客户 battlecard（转 account-research）

## 输出格式

```markdown
## 给【客户称呼/公司名】的行业资讯分享

【称呼】好，最近看到几条和【客户行业/业务】相关的资讯，供您参考：

1. **【一句话洞察】**（《文章标题》｜公众号名｜发布日期）  
   来源：[链接](URL)

2. **【一句话洞察】**（《文章标题》｜公众号名｜发布日期）  
   来源：[链接](URL)

3. **【一句话洞察】**（《文章标题》｜公众号名｜发布日期）  
   来源：[链接](URL)

如果您对其中某个方向感兴趣，我们可以进一步聊聊如何应用到贵司的【业务环节】。
```

## 铁律

- **只引用真实检索结果**：每条资讯必须来自 `search_industry_news` 的返回，禁止编造文章或链接。
- **必须带来源**：标注文章标题、公众号、发布日期和原文链接，方便销售复核。
- **控制长度**：整段文案控制在 300-500 字，适合微信/邮件转发。
- **客户视角**：不要泛泛而谈，要说明「这条资讯对客户意味着什么」。
- **不输出敏感内部信息**。
