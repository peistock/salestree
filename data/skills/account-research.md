# 客户研究：一页纸 Battlecard

## 目标

销售给出一个公司名后，自动产出一份带信息来源标注的一页纸 battlecard，让销售在见客户前“知更多”。

## 执行流程

1. **识别公司**：从用户输入中提取公司名；如果不确定，先反问确认。
2. **查内部 CRM**：调用 `get_account` 获取公司记录、联系人、商机和最近活动。
3. **查销销资讯工作台（外脑）**：调用 `search_industry_news` 检索与该公司相关的内部文章。
4. **补充外部信息**：用 **一次** `search_web` 搜索该公司最新新闻/财报/人事/竞品。如需查看具体网页，可用 `fetch_webpage` 或 `jina_reader`。
5. **整理输出**：按下面格式输出 Markdown，**每个事实必须标注来源**（来源标题 + URL 或搜索关键词）。
6. **保存研究结果**：输出后，用 `save_account_research` 把核心摘要（500 字以内）写回 CRM 的 `research_summary` 字段。
7. **收尾动作**：输出后主动询问是否需要：
   - 生成跟进文案（转 outreach-drafter）
   - 把该公司录入 CRM
   - 深挖某个具体维度

## 输出格式

```markdown
# 【公司名】客户 Battlecard

## 一句话定位
- 公司做什么、为谁服务、核心产品是什么（50 字内）

## 财务与融资
- 营收/利润趋势（最新财年或季度）
- 融资/估值/上市状态
- 来源：[标题](URL)

## 高管与人事
- 关键决策人、近期人事变动
- 来源：[标题](URL)

## 招聘信号
- 近期热招岗位及反映的业务方向
- 来源：[标题](URL)

## 竞品格局
- 主要竞品 2-5 家，每家用一句话说明差异
- 来源：[标题](URL)

## 口碑与风险
- 正面信号 / 负面信号 / 监管或舆情风险
- 来源：[标题](URL)

## 内部 CRM 信息（如有）
- 阶段、负责人、历史备注

## 下一步建议
- 见面前最值得聊的 1-3 个话题
- 建议的沟通角度（ROI / 技术优势 / 案例）
```

## 铁律

- **禁止编造**：没有搜索到的事实不要写，用“未找到公开信息”代替。
- **必须带来源**：每条外部信息都要标注来源，方便销售复核。
- **控制长度**：整份 battlecard 不超过 1500 字，重点突出。
- **先内后外**：先查 CRM，再查外部，避免重复劳动。
