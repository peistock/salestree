# 投流方案 Skill

## 1. 定位

给新客户输出**第一版可交付的投流方案 PPT/报告**。不是最终执行计划，而是把「客户是谁、卖什么、投多少、先投谁、怎么度量」对齐到同一套语境里，让策划和投手能接力。

当客户提供的信息不完整时，本 skill 用行业 benchmark 和三档预算假设先出草稿，并明确标注「待确认」。

## 2. 核心不变式

- **语境 = 角度 × 人群**（对角线匹配，非笛卡尔积）
- **targeting sets ⊂ context**（定向组合嵌套进语境）
- **先验押注，嵌套去跑**（花钱前靠逻辑排序，不是花完删差的）
- **结构刚性，参数弹性**（嵌套逻辑不变；维度/深度/押注数随预算调）
- **策划画框，投手摆棋**（策划定 context，投手填 targeting sets）
- **平台人群代号不混用**（小红书人群标签 ≠ 抖音云图人群代号，输出时按平台选择术语体系）

## 3. 信息缺口诊断

收到需求后，先检查这 7 项：

| 字段 | 是否必需 | 缺失时的处理 |
|---|---|---|
| 客户/品牌 | 必需 | 无法开工，反问 |
| 产品 | 必需 | 无法开工，反问；或从品牌反推主推品 |
| 平台 | 建议有 | 默认客户已在投放，反问目前在跑哪个平台；不替客户做平台选择论证 |
| 预算 | 最常缺失 | 用 benchmark 推三档方案 |
| 周期 | 建议有 | 默认按季度 / 按大促节点 |
| TA | 建议有 | 从产品 + 品类反推假设 TA |
| 卖点/角度 | 建议有 | 从产品资料 + 竞品反推 |

## 4. 预算缺失时的标准打法

不追问预算，直接按品类 benchmark 输出三档：

| 档位 | 预算 | 适合场景 | 预期产出 | 风险 |
|---|---|---|---|---|
| 试投档 | 5–10 万 | 验证 context | 跑通 1-2 个 context | 数据量小，难下结论 |
| 标准档 | 20–50 万 | 新品/常规 campaign | 2-4 个 context 轮换 | 可出稳定洞察 |
| 放量档 | 100 万+ | 大促/破圈 | 多 segments 全链路覆盖 | 需配合内容与 KOL |

用行业 benchmark 反推 KPI（以小红书母婴/护肤类目为例，保守口径）：

| 指标 | 参考范围 |
|---|---|
| CPM | ¥60–120 |
| CPC | ¥0.6–1.5 |
| CPUV | ¥1.0–3.0 |
| CPE | ¥3–15 |

以 20 万预算为例：
- 曝光 ≈ 200–300 万
- 点击 ≈ 15–25 万
- UV ≈ 7–15 万

所有基于假设的数字必须标 `[假设，待确认]`。

## 5. 其他关键信息缺失处理

**TA 缺失**
- 从产品使用场景反推：婴童面霜 → 0-3 岁宝妈
- 给 2-3 个假设 segments，标 `[假设]`
- 例：「若客户未给 TA，本方案按 敏宝妈妈 / 新手妈妈 / 成分妈妈 三 segments 展开，需确认」

**卖点/角度缺失**
- 用 产品功能 → 场景 → 情感 → 背书 四层拆解
- 给 2-3 个候选角度，标 `[待选]`
- 例：「一抹降燥 / 专研胚米 · 大牌放心 / 雪顶奶冻 · 不抗拒」

**平台缺失**
- 默认客户已经在投放，不替客户做「优先做哪个平台」的论证
- 输出时写：「未指定平台，本方案按客户当前在跑平台假设为 {{platform_assumption}}，请确认」
- 若完全无法判断，把平台选择列入待确认清单，而不是自动默认小红书或抖音

**周期缺失**
- 默认按季度（3 个月）
- 或按接下来最近的大促节点：双11 / 618 / 年货节

## 6. 情报收集（新客户必做）

在动手写方案前，先调用工具把能查到的情报查一遍。信息不全也要查，查不到再标「未知」。

### 6.1 产品信息（必须做扎实）

把产品当成一个"待投放的 SKU"来拆，不能只查一句话卖点。至少从以下 6 个维度搜：

**搜索源建议（按优先级）**
- 品牌官网 / 官方公众号：查产品定位、官方话术、技术背书
- 天猫 / 京东旗舰店：查 SKU、规格、价格、促销、用户评价高频词
- 小红书：查真实用户笔记、痛点、使用场景、视觉风格
- 知乎 / 什么值得买 / 垂直媒体：查测评、参数对比、口碑
- 抖音电商 / 直播间：查主播话术、爆款 SKU、价格机制

**提取字段（输出时尽量填全）**

| 字段 | 查什么 | 示例 |
|---|---|---|
| 产品名/SKU | 主推款、辅助款、规格 | 云溪冰箱 635L / 启初胚米面霜 40g |
| 品类位置 | 所属类目、品类心智 | 高端嵌入式冰箱 / 婴童舒缓面霜 |
| 价格带 | 官方价、活动价、套装价 | ¥8999 / ¥89 |
| 核心功能 | 解决什么具体问题 | 全空间保鲜 / 一抹降燥 |
| 成分/技术 | 差异化技术点、专利、背书 | AI直驱双擎 / 胚米精华 |
| 使用场景 | 谁在什么场景用 | 母婴囤货 / 新家装修 |
| 目标用户 | 官方 TA 与真实 TA | 精致妈妈 / 养宠家庭 |
| 核心卖点 | 3 个以内、按优先级 | 自然鲜、零嵌美学、大容量 |
| 视觉/包装 | 风格、IP、色系 | 治愈绿、极简白、联名款 |
| 渠道分布 | 线上/线下、主战场 | 天猫官旗 + 抖音自播 |
| 用户评价关键词 | 好评/差评高频词 | "保鲜好" / "颜值高" / "价格贵" |
| 与上一代/竞品差异 | 升级点、替代关系 | 比上一代容量 +100L |

**调用方式**
- `search_web` 至少 3–5 个不同角度：
  - 「品牌 + 产品 + 官网」
  - 「品牌 + 产品 + 小红书 / 测评」
  - 「品牌 + 产品 + 知乎 / 怎么样」
  - 「产品 + 天猫 / 京东 / 价格」
  - 「产品 + 用户评价 / 缺点」
- `fetch_webpage` / `jina_reader` 读取官网、电商详情页、核心测评页
- 小红书/抖音等需要动态页面的，优先用 `search_web` 摘要 + 网页快照，必要时调用 `browse_open`

### 6.2 历史营销情况（必须做扎实）

不只查有没有投过广告，要还原出"之前打了什么牌、效果可能如何"。

**搜索方向**
- 品牌 campaign：「品牌 + campaign / 营销 / 广告 / 品牌片 / slogan」
- 平台动作：「品牌 + 小红书 / 抖音 / 巨量云图 / 灵犀 / 双11 / 618」
- 明星/KOL：「品牌 + 明星 / 达人 / KOL / 代言 / 合作」
- 话题/内容：「品牌 + 话题 / 挑战赛 / UGC / 直播」
- 品牌资产：「品牌 + 官方账号 / 自播 / 旗舰店 / 内容矩阵」

**提取字段（输出时尽量填全）**

| 字段 | 查什么 |
|---|---|
| campaign 主题 | 近 1–2 年核心传播主题 |
| 时间节奏 | 大促 / 上新 / 节点 |
| 主推 SKU | 每次 campaign 打什么品 |
| 平台组合 | 小红书 / 抖音 / 微博 / 微信 / B站 |
| 内容角度 | 功能痛点 / 情感场景 / 背书 / 价格机制 |
| KOL 结构 | 明星 × 头部 × 中腰部 × KOC 占比 |
| 达人名单（可列举） | 合作过的代表性达人 |
| 话题/标签 | #话题、小蓝词、搜索词 |
| 竞价点位痕迹 | TopView / 开屏 / 信息流 / 搜索 / 千川 / 品专 |
| 直播/自播 | 官方直播间、达人直播间、日不落直播 |
| 公开效果信号 | 互动量、销量排名、搜索指数、媒体报道 |

### 6.3 竞品分析（必须做扎实）

竞品不是只列名字，要找到"客户可以借势或错位打"的机会点。

**确定竞品的方法**
- `search_web`：「品类 + 排行榜 / 推荐 / 十大品牌」
- `search_web`：「品牌 + 竞品 / 对比 / 哪个好」
- `search_web`：「小红书 + 品类 + 测评」
- `search_web`：「知乎 + 品类 + 怎么样 / 推荐」
- 资讯看板 `search_industry_news`：按品类搜行业文章，看高频提及品牌

**分析维度（每个竞品填一份）**

| 维度 | 查什么 |
|---|---|
| 品牌名 + 核心 SKU | 直接对标款 |
| 价格带 | 与本品相比贵/便宜多少 |
| 定位 | 高端 / 性价比 / 专业 / 情感 |
| 目标人群 | 与本品重叠或差异 |
| 核心卖点 | 技术 / 成分 / 场景 / 背书 |
| 近期 campaign | 近 1 年主题、平台、角度 |
| KOL/内容策略 | 头部测评、母婴/生活/美妆垂类 |
| 平台主战场 | 小红书 / 抖音 / 天猫 / 京东 |
| 强势点 | 客户可能难以正面打的 |
| 弱势/空白点 | 客户可以切入的机会 |

**输出要求**
- 至少找 3 个核心竞品，最好是 1 个直接竞品 + 1 个高端参照 + 1 个性价比参照
- 最后给出「本品 vs 竞品」一句话结论：我们能在哪个语境上打出差异

### 6.4 资讯看板 / 知识库

- `get_news_digest`：读取资讯看板整体摘要，看行业近期热点
- `search_industry_news`：按品类/关键词搜索相关文章（如「婴童面霜」「精华水」「家电种草」）
- 提取：品类趋势、竞品玩法、可借鉴的内容角度、潜在机会点

### 6.6 投流方法论页（客户产品分析后必讲）

在讲客户具体方案之前，会先由系统自动注入《定向框架 — 统一投手与策划》10 页方法论幻灯片。第一页标题为「亿科投流方法论 · 定向框架」，其余 9 页内容原封不动，只做配色/样式与整体方案统一。LLM 无需在正文生成这部分。

## 7. 工作流程

**Step 0 · 情报收集**
- 按 6.1–6.5 调用工具，产品、历史营销、竞品三个模块必须做扎实
- 每个模块至少调用 3–5 次不同角度的 `search_web`，配合 `fetch_webpage` / `jina_reader` 读详情页
- 查不到的信息标「未知」，不编造

**Step 1 · 前策消化 + 方法论对齐**
- 读 brief、前策 PPT、数据表（如有）
- 把情报摘要和 brief 做交叉验证
- 揪错漏、串稿、数据矛盾
- 输出《前策阅读概要》
- **输出《投流方法论页》（Page A + Page B），让客户先理解我们的逻辑和策划×投手协作方式**

**Step 2 · 分品定位**
- 一条时间线，几款产品接力
- 每款定：推广期、传播目的、核心预算
- 例：高保湿霜 7-8 月降燥起量 → 特润霜 9-12 月秋冬滋养 + 大促收割

**Step 3 · 预算 → 产品 → TA → segments**
- 钱先落到产品
- 产品定 TA
- TA 切成可操作的 segments
- **按平台选人群术语体系**：小红书用小红书圈层语言（如精致妈妈、成分党、新手妈妈），抖音用巨量云图人群代号（如精致妈妈、资深中产、新锐白领），禁止两套术语混用

**Step 4 · 配语境 context = 角度 × 人群**
- 给每个 segment 配对内容角度
- 例（小红书）：敏宝妈妈 × 一抹降燥；成分妈妈 × 专研胚米
- 例（抖音）：精致妈妈 × 自然鲜享受世界；养宠家庭 × 零嵌岩感美学

**Step 5 · 拆 targeting sets**
- 在 context 内组合需求 × 消费 × 决策
- 维度可坍缩：小预算 1-2 维，大预算 3 维
- **全域投放适配（抖音）**：
  - 不再写死「人群包」，而是把 targeting set 拆成「素材角度 + 转化目标 + 出价策略」的组合
  - 一个 context 对应 1–3 个全域计划：不同角度用不同素材组，同一角度用不同转化目标/出价做赛马
  - 例：新手妈妈省心 → 素材组 A（带娃场景） + 优化成交 + 控成本；素材组 B（专家背书） + 优化商品点击 + 放量

**Step 6 · 选点位（平台-specific）**
- 小红书：信息流、搜索、全站、视频流
- **抖音（当前以全域投放为主）**：
  - 千川全域推广是主要竞价点位，算法自动探索人群
  - 内容场弹药通过「素材角度 + 转化目标 + 预算分配」喂给全域计划
  - 搜索/小蓝词/品专承接看后搜与品牌收割
- 原则：每个内容配一个竞价策略去承接，不裸奔

**Step 7 · 先验押注 + 节奏**
- 从 N 个组合里押最可能赢的 1-3 个
- 排期：蓄水 → 种草 → 收割
- 效果类晚于种草 1 个月起投

**Step 8 · 关键词 / 搜索策略**
- 按购买路径编组：场景痛点 → 品类比较 → 竞品截流 → 品牌收割
- 泛搜养认知，精搜抢决策

**Step 9 · KPI + 单位成本护栏**
- 单位成本 = 总预算 ÷ 总产出（不许取子级平均）
- 输出：曝光、CPM、点击、CPC、UV、CPUV、互动、CPE
- 加反算自检

**Step 10 · 操盘 SOP**
- 赛马 → 拓量 → 稳效
- 60/20/20 滚动换新
- 日粒度 actual vs plan，三级效能矩阵

**Step 11 · 待确认清单**
- 把不能由 AI 拍板的判断列出来交给客户/销售确认

## 二、输入说明

| 字段 | 当前处理 | 真实值 |
|---|---|---|

## 10. 铁律

1. **第一页必须加免责声明**：所有数字仅为展示逻辑与测算口径，不代表执行效果，最终以后台实测为准
2. **客户产品分析后必须先输出方法论页**：让客户理解「品→效→销」三层逻辑和策划×投手协作方式，再进入具体方案
3. **不许编造预算**：没有预算就输出三档假设方案
4. **所有假设必须可反算**：单位成本 = 总 ÷ 总，数字能回源
5. **先出草稿再确认**：不完备信息下，方案价值是「帮销售把问题问全」
6. **多档优于单档**：客户没给预算时，给三档比给一个更稳
7. **对角线匹配**：角度和人群必须成对，不穷举
8. **先验押注**：花钱前排序，不是花完删
9. **效果类晚于种草**：人群池养厚再开闸
10. **内容不裸奔**：每个 KOL/IP 配一个竞价策略/计划去承接、放大（抖音全域下即一个素材角度 + 转化目标组合）
11. **AI 只做到 80%**：判断悬崖边交给人，输出草稿 + 待确认清单
12. **查不到的情报标「未知」**：不编造搜索不到的信息
13. **产品 / 历史营销 / 竞品必须做扎实**：每个模块至少调用 3–5 次不同角度的搜索，不能只靠一次搜索出结论
14. **竞品分析要找机会点**：不只是列竞品，要给出「本品能错位打哪里」
15. **小红书与抖音人群代号不混用**：输出 segments、context、targeting sets 时，必须按平台使用对应术语体系
16. **A1-A5 是抖音千川/云图的 5A 标签，不是小红书标签**：小红书用认知 / 兴趣 / 种草 / 购买 / 分享，禁止把 A1/A2/A3 套到小红书方案上
17. **默认客户已经在投放，不做平台选择论证**：不要输出「为什么优先做抖音/小红书」这类页；若平台未指定，反问客户目前在跑哪个平台
