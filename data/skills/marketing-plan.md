# 整合营销方案生成器

## 定位（通用框架）

当前按「整合营销服务公司」搭框架，服务线覆盖：
- **品牌策略**：品牌定位、价值主张、视觉体系、IP 打造
- **内容营销**：品牌故事、白皮书、案例内容、SEO 内容
- **社媒运营**：双微一抖小红书/B 站等内容矩阵与日常运营
- **效果广告**：信息流、搜索广告、效果归因与投放优化
- **短视频 / 直播**：创意脚本、达人合作、直播间运营
- **KOL / KOC**：达人筛选、种草矩阵、口碑管理
- **私域运营**：企微/社群运营、会员体系、复购提升
- **电商运营**：平台入驻、详情页优化、活动策划

> 后续可替换为具体公司服务线与案例库。

## 目标

销售给出一个客户（或业务目标）后，销销输出一份可直接用于内部讨论或客户提案的营销方案 Markdown，包含：客户洞察、目标拆解、策略、执行、预算框架、效果指标。

## 执行流程

1. **识别客户与需求**
   - 从用户输入中提取：客户公司名、所属行业、当前业务目标、预算级别（如有）、时间周期。
   - 如信息缺失，先简短反问（最多 1-2 个问题）。

2. **加载客户信息**
   - 用 `get_account` 查 CRM 中是否有该客户；若 `research_summary` 字段存在，直接复用其中的纵横分析结论。
   - 如无研究摘要，用 `search_web` 搜索该公司近期业务动态、产品动作、市场表现、营销动作。
   - 用 `search_industry_news` 在 wechat-digest 资讯库中搜索行业趋势、标杆案例、平台政策，为方案提供一手行业洞察。
   - 用 `search_web` 搜索行业趋势 2-3 条。

3. **选择核心策略方向**
   根据客户阶段和目标，从以下方向中选择 1-2 个主策略 + 1-2 个支撑策略：
   - **品牌势能型**：适合新品上市、品牌升级、融资后发声
   - **效果增长型**：适合获客成本高、需要快速拿量
   - **内容种草型**：适合决策周期长、需要信任沉淀
   - **私域复购型**：适合已有用户池、需要提升 LTV
   - **电商转化型**：适合大促节点、需要短期 GMV

4. **输出方案**
   按下面格式输出 Markdown，**策略必须结合客户事实**，禁止泛泛而谈。

5. **收尾动作**
   输出后询问是否需要：
   - 生成一份 PPT 版提案（转 ppt-master）
   - 生成跟进邮件/微信触达文案（转 outreach-drafter）
   - 深挖客户 battlecard（转 account-research）
   - 调整预算/策略方向

## 输出格式

```markdown
# 【客户公司】整合营销方案

## 1. 客户洞察
- 公司定位与核心业务（50 字内）
- 当前业务目标/挑战（如：新品上市、获客降本、品牌升级）
- 目标人群画像（B2B / B2C / 决策链）
- 近期关键动态（来源：[标题](URL)）

## 2. 营销目标（SMART 化建议）
- 品牌目标：知名度/认知度/美誉度
- 效果目标：线索数 / CAC / ROI / GMV
- 时间节点：建议分阶段（如 0-30 天 / 1-3 个月 / 3-6 个月）

## 3. 核心策略
- 主策略 1：……（为什么选这个）
- 主策略 2：……
- 支撑策略：……

## 4. 执行计划

### 阶段一：预热 / 起势（0-30 天）
- 关键动作
- 内容/达人/渠道组合
- 预期产出

### 阶段二：放量 / 转化（1-3 个月）
- 关键动作
- 内容/达人/渠道组合
- 预期产出

### 阶段三：沉淀 / 复购（3-6 个月）
- 关键动作
- 内容/达人/渠道组合
- 预期产出

## 5. 预算框架（按通用营销公司口径）
| 项目 | 占比 | 说明 |
|------|------|------|
| 策略与内容 | 15%-20% | 品牌定位、创意、脚本、文案 |
| 媒介投放 | 40%-50% | 信息流、搜索、开屏、达人采买 |
| 达人/KOL | 15%-25% | 种草、直播、背书 |
| 运营执行 | 10%-15% | 社媒、私域、社群、客服 |
| 技术与数据 | 5%-10% | 落地页、数据看板、归因工具 |

> 注：具体预算需根据客户实际体量、目标和周期重新测算。

## 6. 效果指标
- **过程指标**：曝光、互动、CTR、CPM、CPV
- **结果指标**：线索成本、转化率、ROI、GMV、品牌搜索指数
- **复盘节奏**：周报/月报/阶段复盘

## 7. 风险与应对
- 风险 1：……
- 应对：……

## 8. 下一步建议
- 本周内可推进的 3 个动作
- 需要客户确认的事项
```

## 铁律

- **不写没依据的策略**：每个策略必须基于客户事实或行业趋势，禁止万能模板。
- **预算只说框架**：具体金额用区间表示，避免在信息不全时给出具体报价。
- **可执行**：每个阶段必须有明确动作和产出，不能只有愿景。
- **来源标注**：客户动态、行业趋势必须标注来源。
- **强制格式**：每条关键事实后必须紧跟来源，格式如 `事实描述（来源：[标题](URL)）`，不要只在文末写"公开信息整理"。
