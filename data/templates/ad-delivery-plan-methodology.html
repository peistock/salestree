Methodology M02

为什么要统一 · The Gap

策划想「内容」，投手想「定向」——两张皮

素材在讲一套故事，定向在框另一群人，中间没人对账。

策划视角
  • 讲什么故事、什么角度
  • 内容 / 素材 / 卖点
  • 想的是「打动谁的心」
投手视角
  • 框哪群人、怎么出价
  • 人群包 / 定向 / 出价
  • 想的是「投得中谁」
结果：素材讲一套、定向框另一套，素材和人对不上，预算在两套逻辑之间空转。

Methodology M04

推导·上游 · Budget → People

预算的第一约束是「产品」，不是「人群」

钱先落到推哪个产品，产品才定谁会买，TA 再切成一个个人群段。

1. 钱从哪来、投多少
一切的起点
2. 推哪个产品
预算的第一个约束
3. 谁会买这个产品
产品决定受众
4. 把 TA 切成一个个人群段
可操作的「谁」
到这里只解决了「谁」——还差最关键的一跳：把「谁」和「角度」配成语境。
Methodology M05

关键一跳 · The Pairing

context = 角度 × 人群，成对绑定

投放的原子单位是语境——同一张床单，卖美学就讲设计，卖舒适就讲材质。

角度＼人群美学人群舒适人群
讲设计（角度）✓对味✕错配
讲材质（角度）✕错配✓对味
× 不是笛卡尔积（全排列），是 配对 ——只有对角线成立。策划管的就是这一层：给对的人，配对的角度。
Methodology M06

推导·下游 · Refine

在一个语境里，把人群再细分成定向组合

需求 × 消费 × 决策…… 组合出一批 targeting sets——这是投手在框里摆的棋。

一个语境
context = 讲设计 × 美学人群
需求 A1 × 消费 B1 × 决策 C1
A1 × B2 × C2
A2 × B1 × C2
A2 × B3 × C1
A3 × B2 × C2 … 共 N 个组合
context 是策划画的框，targeting sets 是投手在框里摆的棋 ——维度不是固定的，下一页讲弹性。
Methodology M07

核心动作 · Prior not Posterior

先验预判，不是后验筛选

一分钱没花、没有数据，靠逻辑先押可能赢的几个，放进语境去 run。

✓ 这才是 match
  • 花钱前，靠判断给组合排序
  • 从 N 个里押最可能赢的几个
  • 放进语境去 run
✕ 这不是 match
  • 后验筛选
  • 花完钱，看数据删掉差的
  • 先撒 27 个，再事后砍——这是交学费，不是判断
先押、后验证，不是先撒、后删——省下的就是那笔学费。
Methodology M08

框架内核 · Rigid & Elastic

结构刚性，参数弹性

嵌套逻辑恒定不变；用几维、下探多深、押几个，都随实战条件现场调档。

不变式：targeting sets ⊂ context —— 恒定不变，不商量
旋钮 1 · 定向维度
3 维（需求×消费×决策） → 2 维 → 1 维
由预算 / 数据 / 品类决定
旋钮 2 · 语境深度
context → 要不要下探 sub context
由预算量级 / 周期长短决定
旋钮 3 · 押几个
27 选 3 / 9 选 X / 3 选 1
组合随维度缩放，押的动作不变
维度可以坍缩，嵌套不能坍缩。换个预算，27 选 3 就成 3 选 1，但它仍是同一个框架。
